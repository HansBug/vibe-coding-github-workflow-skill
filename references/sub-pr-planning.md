# Sub PR Planning (WIP)

This reference will define how large vibe coding tasks are decomposed into sub PRs while keeping the parent issue or umbrella PR readable.

Planned scope:

- decide when a task needs sub PRs instead of a single implementation PR;
- maintain task tables with PR id, goal, dependencies, deliverables, exit gate, and status;
- maintain Mermaid dependency graphs and update them after each sub PR lands or changes scope;
- record sub PR merge summaries without losing the parent plan's current state.

This file is intentionally a placeholder until the FS-B/FS-C phases expand the template and examples.
